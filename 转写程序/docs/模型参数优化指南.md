# FunASR 模型参数优化指南

## 📋 概述

本文档介绍如何调整 FunASR 模型参数以提高语音识别的准确度、稳定性和性能。

## 🎯 当前配置

### 1. 模型初始化参数


## 🔧 可调整参数详解

### 一、VAD（语音活动检测）参数

VAD 用于区分语音和非语音部分，影响识别的准确性和完整性。

#### 1. `max_single_segment_time` (默认: 15000ms)

**说明：** 单个语音片段的最大时长（毫秒）

**建议：**
- **短语音/对话：** 6000-10000 (6-10秒)
- **长语音/演讲：** 15000-30000 (15-30秒)
- **默认推荐：** 15000

```python
self.asr_model = AutoModel(
    model="paraformer-zh",
    vad_model="fsmn-vad",
    vad_kwargs={
        "max_single_segment_time": 15000  # 15秒
    }
)
```

#### 2. `speech_noise_thres` (默认: 0.6)

**说明：** 语音噪声阈值，范围 0-1，越高越严格

**建议：**
- **低噪环境（录音室）：** 0.5-0.6（识别更多内容）
- **中等噪音：** 0.6-0.7（默认，平衡）
- **高噪环境：** 0.7-0.8（过滤噪音，可能漏掉弱音）

```python
vad_kwargs={
    "speech_noise_thres": 0.6  # 中等阈值
}
```

#### 3. `min_silence_duration_ms` (默认: 500ms)

**说明：** 最小静音持续时间，用于判断语音段结束

**建议：**
- **快速对话：** 200-300ms（避免切断连续语音）
- **正常语速：** 500ms（默认）
- **慢速/有停顿：** 700-1000ms（更明确的分段）

```python
vad_kwargs={
    "min_silence_duration_ms": 500
}
```

### 二、批处理参数

#### `batch_size_s` (默认: 300秒)

**说明：** 批处理大小，影响内存使用和处理速度

**建议：**
- **短音频 (<1分钟)：** 60-120秒
- **中等音频 (1-5分钟)：** 300秒（默认）
- **长音频 (>5分钟)：** 600-1800秒
- **内存有限：** 减小到 60-120秒

```python
asr_result = self.asr_model.generate(
    input=audio_url,
    batch_size_s=300  # 5分钟批处理
)
```

### 三、热词（Hotword）优化

#### 作用

热词可以提高特定词汇的识别准确率，适用于专业术语、人名、地名等。

#### 格式

```python
# 单个热词
hotword='阿里云'

# 多个热词（用空格分隔）
hotword='阿里云 魔搭 FunASR 语音识别'

# 带权重的热词（推荐）
hotword='阿里云 20 魔搭 15 FunASR 10'
# 权重越高，识别优先级越高
```

#### 示例配置

```python
# 会议场景
hotword='项目管理 20 需求分析 15 技术方案 15 系统架构 10'

# 医疗场景
hotword='高血压 20 糖尿病 20 心电图 15 CT 15 核磁共振 10'

# 电商场景
hotword='订单号 20 物流信息 15 售后服务 15 退款 10 评价 10'
```

### 四、说话人识别参数

#### 1. `threshold` (默认: 0.6)

**说明：** 声纹识别相似度阈值，范围 0-1

**建议：**
- **严格识别（少误判）：** 0.7-0.8
- **平衡模式：** 0.6-0.7（默认推荐）
- **宽松识别（少漏判）：** 0.5-0.6

```python
result = system.diarize_with_auto_registration(
    audio_path,
    threshold=0.65  # 稍微严格一点
)
```

### 五、高级参数

#### 1. `beam_search` (波束搜索)

**说明：** 影响解码策略

```python
# 在模型 generate 时添加
asr_result = self.asr_model.generate(
    input=audio_url,
    batch_size_s=300,
    beam_size=5,        # 波束大小，默认10
    # beam_size 越大：准确度可能更高，但速度更慢
    # beam_size 越小：速度更快，但可能损失准确度
)
```

**建议：**
- **快速识别：** beam_size=3
- **平衡模式：** beam_size=5-10（默认）
- **高精度：** beam_size=10-15

#### 2. `decoding_chunk_size`

**说明：** 解码块大小，影响实时性和准确性

```python
asr_result = self.asr_model.generate(
    input=audio_url,
    decoding_chunk_size=-1,  # -1 表示离线模式（最准确）
    # 正数表示流式模式（实时，但可能略降低准确度）
)
```

**建议：**
- **离线处理（最准确）：** -1（默认推荐）
- **实时处理：** 4-8

## ⚠️ 注意事项

### 1. 参数调整原则

- ✅ **逐步调整：** 一次只改一个参数，观察效果
- ✅ **记录结果：** 保存每次调整的参数和效果
- ✅ **多次测试：** 用多个音频测试，避免偶然性
- ⚠️ **权衡取舍：** 准确度、速度、资源使用需要平衡

### 2. 常见错误

- ❌ **阈值过高：** 导致漏掉真实语音
- ❌ **阈值过低：** 导致噪音被识别为语音
- ❌ **批处理过大：** 内存溢出
- ❌ **批处理过小：** 处理效率低

### 3. 优化建议

- 📊 **先测试默认参数：** 建立基准
- 🎯 **针对场景优化：** 不同场景用不同配置
- 📝 **保存配置文件：** 便于切换和管理
- 🔄 **定期评估：** 随着数据变化调整参数

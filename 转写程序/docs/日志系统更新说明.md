# 日志系统更新说明

## 更新内容

已将所有 `print` 输出改为使用日志系统 (`logging` 模块)，所有日志会自动保存到文件中。

## 主要变更

### 1. 新增文件

- `logger_config.py` - 日志配置模块
- `logs/` - 日志目录（自动创建）
- `funasr.service` - 更新的 systemd 服务配置
- `日志配置说明.md` - 详细的日志使用文档

### 2. 修改的文件

- `api_server.py` - 所有 print 改为 logger
- `paraformerdome.py` - 所有 print 改为 logger

## 日志输出位置

所有日志自动保存在 `logs/` 目录：

```
logs/
├── funasr_20251107.log          # 所有日志（INFO, WARNING, ERROR）
├── funasr_error_20251107.log    # 仅错误日志
└── .gitkeep
```

## 日志格式

```
2025-11-07 16:47:46 | INFO     | funasr.api | 正在启动语音识别服务...
2025-11-07 16:47:46 | ERROR    | funasr.asr | 识别失败: Invalid audio format
```

## 快速开始

### 本地开发环境（Windows）

无需额外配置，直接启动即可：

```bash
# 激活虚拟环境
venv\Scripts\activate

# 启动服务
python api_server.py
```

日志会自动输出到：
- 控制台（实时查看）
- 文件 `logs/funasr_YYYYMMDD.log`

### 生产环境（CentOS/Linux）

#### 方式1：直接运行

```bash
# 激活虚拟环境
source venv/bin/activate

# 后台运行
nohup python api_server.py > /dev/null 2>&1 &

# 查看日志
tail -f logs/funasr_$(date +%Y%m%d).log
```

#### 方式2：使用 systemd 服务

```bash
# 复制服务文件
sudo cp funasr.service /etc/systemd/system/

# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start funasr

# 查看服务状态
sudo systemctl status funasr

# 查看应用日志
tail -f /opt/speechtotext_server/logs/funasr_$(date +%Y%m%d).log

# 查看系统服务日志
sudo journalctl -u funasr -f
```

## 日志级别说明

| 级别    | 用途                         | 示例                       |
|---------|------------------------------|----------------------------|
| DEBUG   | 调试信息（默认不记录）       | 临时文件保存成功           |
| INFO    | 一般信息                     | 服务启动、模型加载完成     |
| WARNING | 警告（不影响功能）           | 临时文件删除失败           |
| ERROR   | 错误（需要关注）             | 识别失败、模型加载失败     |

## 查看日志

### Linux/CentOS

```bash
# 实时查看最新日志
tail -f logs/funasr_$(date +%Y%m%d).log

# 查看错误日志
tail -f logs/funasr_error_$(date +%Y%m%d).log

# 搜索特定内容
grep "识别完成" logs/funasr_*.log

# 查看最后100行
tail -n 100 logs/funasr_$(date +%Y%m%d).log
```

### Windows PowerShell

```powershell
# 实时查看日志
Get-Content logs\funasr_$(Get-Date -Format "yyyyMMdd").log -Tail 50 -Wait

# 搜索错误
Select-String "ERROR" logs\funasr_*.log

# 查看最后100行
Get-Content logs\funasr_$(Get-Date -Format "yyyyMMdd").log -Tail 100
```

## 日志管理

### 日志轮转

- 单个文件最大 **50MB**
- 保留 **10个** 备份文件
- 自动按日期分文件
- 超过大小自动创建新文件（如 `.log.1`, `.log.2`）

### 日志清理

建议定期清理旧日志：

```bash
# 删除7天前的日志（Linux）
find logs/ -name "*.log*" -mtime +7 -delete

# 删除30天前的日志
find logs/ -name "*.log*" -mtime +30 -delete
```

可以添加到 crontab 定时任务：

```bash
# 编辑定时任务
crontab -e

# 添加：每天凌晨2点清理30天前的日志
0 2 * * * find /opt/speechtotext_server/logs/ -name "*.log*" -mtime +30 -delete
```

## 配置修改

如需修改日志配置，编辑 `logger_config.py`：

```python
# 修改日志级别（启用调试日志）
logger = setup_logger(log_level=logging.DEBUG)

# 修改日志目录
logger = setup_logger(log_dir="/var/log/funasr")

# 修改单个文件大小限制
maxBytes=100*1024*1024  # 100MB
```

修改后需要重启服务：

```bash
# systemd 方式
sudo systemctl restart funasr

# 直接运行方式
# 按 Ctrl+C 停止，然后重新启动
python api_server.py
```

## 常见问题

### Q1: 控制台中文乱码

**A:** 这是 Windows CMD/PowerShell 的编码问题，不影响日志文件。日志文件中的中文是正常的（UTF-8编码）。

### Q2: 日志文件占用空间太大

**A:** 可以：
1. 减小单个文件大小限制（修改 `logger_config.py`）
2. 减少备份文件数量
3. 定期清理旧日志（建议保留7-30天）

### Q3: 如何只查看错误日志

**A:** 直接查看错误日志文件：

```bash
tail -f logs/funasr_error_$(date +%Y%m%d).log
```

或从主日志中筛选：

```bash
grep "ERROR" logs/funasr_*.log
```

### Q4: 如何临时启用详细日志

**A:** 修改 `logger_config.py` 中的日志级别：

```python
# 改为 DEBUG 级别
logger = setup_logger(log_level=logging.DEBUG)
```

重启服务后生效。

## 性能影响

- 日志记录对性能影响很小（< 1%）
- 异步写入，不阻塞主线程
- 建议生产环境使用 INFO 级别
- 仅在调试时使用 DEBUG 级别

## 部署检查清单

部署到生产环境前，请确认：

- [ ] `logs/` 目录存在且有写入权限
- [ ] systemd 服务配置已更新（`funasr.service`）
- [ ] 日志目录有足够磁盘空间（建议 10GB+）
- [ ] 已配置日志自动清理（crontab）
- [ ] 测试日志能正常写入
- [ ] 确认日志级别为 INFO（不是 DEBUG）

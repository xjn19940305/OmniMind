## 🏗️ 模型架构说明

### 完整的识别流程

```
音频输入
   ↓
[VAD模型] - 检测语音片段，过滤静音和噪音
   ↓
[ASR模型] - 将语音转为文字
   ↓
[标点模型] - 为文本添加标点符号
   ↓
[说话人模型] - 区分不同说话人
   ↓
[声纹模型] - 识别具体说话人身份
   ↓
最终结果：带标点的文本 + 说话人信息 + 时间轴
```

### 当前使用的模型配置

```python
# ASR 模型（集成了多个子模型）
self.asr_model = AutoModel(
    model="paraformer-zh",              # 主识别模型
    model_revision="v2.0.4",
    vad_model="fsmn-vad",               # VAD 模型
    vad_model_revision="v2.0.4",
    punc_model="ct-punc-c",             # 标点模型
    punc_model_revision="v2.0.4",
    spk_model="cam++",                  # 说话人模型
    spk_model_revision="v2.0.2",
)

# 独立的声纹模型
self.sv_model = AutoModel(
    model="iic/speech_campplus_sv_zh-cn_16k-common"
)
```

## 1. ASR主模型（语音识别）

### 作用
将语音信号转换为文字，是整个系统的核心。

### 可选模型详解

#### 🌟 paraformer-zh (推荐)
```python
model="paraformer-zh"
model_revision="v2.0.4"
```

**特点：**
- ✅ 非自回归架构，速度快
- ✅ 支持中文、英文混合
- ✅ 准确率高（WER < 5%）
- ✅ 内存占用适中（~2GB）
- ✅ RTF < 0.1（实时率）

**适用场景：**
- 通用中文识别
- 会议转写
- 字幕生成
- 客服录音

**优势：**
- 平衡性能最好
- 社区支持最好
- 更新维护频繁

---

#### 🚀 paraformer-zh-streaming (实时场景)
```python
model="paraformer-zh-streaming"
```

**特点：**
- ✅ 支持流式识别
- ✅ 低延迟（< 200ms）
- ✅ 适合实时场景
- ⚠️ 准确率略低于离线版

**适用场景：**
- 实时语音转写
- 视频会议实时字幕
- 智能客服
- 语音助手

**使用示例：**
```python
# 流式识别（需要配合流式输入）
chunk_size = [5, 10, 5]  # [5s, 10s, 5s]
encoder_chunk_look_back = 4
decoder_chunk_look_back = 1

result = model.generate(
    input=audio_stream,
    chunk_size=chunk_size,
    encoder_chunk_look_back=encoder_chunk_look_back,
    decoder_chunk_look_back=decoder_chunk_look_back
)
```

## 2. VAD模型（语音活动检测）

### 可选模型详解

#### 🌟 fsmn-vad (推荐)
```python
vad_model="fsmn-vad"
vad_model_revision="v2.0.4"
```

**特点：**
- ✅ 基于 FSMN 架构
- ✅ 速度快（RTF < 0.01）
- ✅ 准确率高（F1 > 95%）
- ✅ 低延迟

**性能指标：**
- 准确率：96.5%
- 召回率：95.8%
- 处理速度：实时 100 倍

---

#### 🎯 fsmn-vad-zh (中文优化)
```python
vad_model="fsmn-vad-zh"
```

**特点：**
- ✅ 针对中文语音优化
- ✅ 更好的中文停顿检测
- ✅ 适合中文长句

**适用场景：**
- 纯中文音频
- 中文演讲
- 中文会议

---

#### 🌍 silero-vad (多语言)
```python
vad_model="silero-vad"
```

**特点：**
- ✅ 支持多语言
- ✅ 轻量级
- ⚠️ 准确率略低

## 3. 标点模型（标点预测）

### 作用
为识别出的文本添加标点符号，提高可读性。

### 可选模型详解

#### 🌟 ct-punc-c (推荐)
```python
punc_model="ct-punc-c"
punc_model_revision="v2.0.4"
```

**特点：**
- ✅ 轻量级（< 100MB）
- ✅ 速度快
- ✅ 支持常用标点
- ✅ 低延迟

**支持的标点：**
- 句号（。）
- 逗号（，）
- 问号（？）
- 感叹号（！）

---

#### 📝 ct-punc (标准版)
```python
punc_model="ct-punc"
```

**特点：**
- ✅ 标准架构
- ✅ 稳定性好

---

#### 💎 punc-ct-transformer (高精度)
```python
punc_model="punc-ct-transformer"
```

**特点：**
- ✅ 更高准确率
- ✅ 支持更多标点类型
- ⚠️ 速度较慢

**适用场景：**
- 对标点要求高的场景
- 正式文档转写
- 出版物

---

#### ⚡ 禁用标点模型
```python
punc_model=None  # 不使用标点模型
```

**优势：**
- ✅ 速度更快
- ✅ 内存占用更小

**劣势：**
- ❌ 输出无标点符号
- ❌ 可读性差

**适用场景：**
- 后续自己处理标点
- 速度优先场景

## ❓ 常见问题

### Q1: 如何选择合适的模型？

**A:** 按以下优先级选择：
1. 先确定场景（通用/实时/高精度）
2. 再考虑资源（内存/CPU）
3. 测试效果，微调参数

### Q2: 切换模型需要重新训练吗？

**A:** 不需要。FunASR 的模型都是预训练好的，直接加载即可使用。

### Q3: 切换模型后识别效果变差？

**A:** 可能原因：
1. 音频质量不适合该模型
2. 参数配置不当
3. 需要调整 VAD 阈值

**解决方案：**
1. 用多个音频测试
2. 参考场景配置调整参数
3. 查看日志找出问题

### Q4: 可以禁用某些模型吗？

**A:** 可以，设置为 `None` 即可：
```python
punc_model=None,  # 禁用标点
spk_model=None,   # 禁用说话人
```

### Q5: 模型在哪里下载？

**A:** 首次使用时自动从 ModelScope 下载，保存在：
- Linux: `~/.cache/modelscope/`
- Windows: `C:\Users\用户名\.cache\modelscope\`

### Q6: 如何查看已下载的模型？

**A:**
```bash
# Linux/Mac
ls ~/.cache/modelscope/hub/

# Windows PowerShell
Get-ChildItem $env:USERPROFILE\.cache\modelscope\hub\
```

### Q7: 模型占用太多磁盘空间？

**A:** 可以删除不用的模型：
```bash
# 删除特定模型
rm -rf ~/.cache/modelscope/hub/模型名称

# 清空所有缓存（会重新下载）
rm -rf ~/.cache/modelscope/
```

### Q8: 可以自己训练模型吗？

**A:** 可以，但需要：
1. 大量标注数据（数千小时）
2. 强大的计算资源（多GPU）
3. 深度学习专业知识

建议使用预训练模型 + 热词优化。

### 2. 快速提升准确度

**问题：**
- ❓ 如何快速提升识别准确率？
- ❓ 专业术语识别不准怎么办？

**答案：** 👉 按优先级实施

**Top 5 方法：**

1. **添加热词** ⭐⭐⭐⭐⭐ (最快见效)
   ```python
   hotword='专业术语 20 关键词 15'
   ```

2. **切换到大模型** ⭐⭐⭐⭐⭐
   ```python
   model="paraformer-large-zh"  # 准确率提升 30%
   ```

3. **调整噪声阈值** ⭐⭐⭐⭐
   ```python
   "speech_noise_thres": 0.5  # 安静环境
   ```

4. **使用高质量音频** ⭐⭐⭐⭐⭐
   - WAV/FLAC 格式
   - 16kHz 采样率

5. **优化分段参数** ⭐⭐⭐⭐
   ```python
   "min_silence_duration_ms": 300  # 快速对话
   ```

### 3. 参数调优

**问题：**
- ❓ 有哪些参数可以调整？
- ❓ 参数的作用是什么？
- ❓ 如何针对我的场景优化？

**答案：** 👉 查看 `参数速查表.md` 或 `模型参数优化指南.md`

**核心参数：**

| 参数 | 默认值 | 作用 | 快速调整建议 |
|------|--------|------|--------------|
| `speech_noise_thres` | 0.6 | 噪声阈值 | 嘈杂↑0.75 安静↓0.5 |
| `min_silence_duration_ms` | 500 | 停顿判断 | 快速↓300 慢速↑800 |
| `hotword` | "" | 热词 | 添加专业术语 |
| `threshold` | 0.6 | 声纹阈值 | 严格↑0.7 宽松↓0.55 |

### 5. 问题诊断

**问题：**
- ❓ 有语音被漏掉了
- ❓ 噪音被误识别
- ❓ 语音被切断
- ❓ 专业术语错误多

**答案：** 👉 查看 `参数速查表.md` 的"问题诊断速查"

**常见问题速查：**

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 语音被漏掉 | 阈值太高 | `speech_noise_thres: 0.6 → 0.5` |
| 噪音误识别 | 阈值太低 | `speech_noise_thres: 0.6 → 0.75` |
| 语音被切断 | 停顿太短 | `min_silence_duration_ms: 500 → 700` |
| 专业术语错误 | 缺少热词 | 添加 `hotword="术语 20"` |

## 🚀 快速开始

### 方法1：添加热词（最简单，最快见效）

```python
# 在 paraformerdome.py 的 generate 调用中添加
asr_result = self.asr_model.generate(
    input=audio_url,
    batch_size_s=300,
    hotword='阿里云 20 魔搭 15 FunASR 10'  # 添加这行
)
```

### 方法2：切换到大模型（提升30%准确率）

```python
# 在 paraformerdome.py 的 __init__ 中修改
self.asr_model = AutoModel(
    model="paraformer-large-zh",  # 改这里
    model_revision="v2.0.4",
    # ... 其他参数不变
)
```

### 方法3：使用预设配置

```python
# 引入配置
from model_config import get_config

# 获取场景配置
config = get_config("high_quality")

# 使用配置参数
self.asr_model = AutoModel(
    model="paraformer-zh",
    vad_kwargs=config['vad_kwargs']  # 使用配置
)
```

## 💡 最佳实践

### ✅ 优化流程

```
1. 基准测试
   ↓
2. 选择合适的模型组合
   ↓
3. 添加行业热词
   ↓
4. 调整VAD参数
   ↓
5. 测试验证
   ↓
6. 微调优化
   ↓
7. 部署应用
```

### ✅ 调优原则

1. **一次改一个参数** - 便于对比效果
2. **记录每次结果** - 建立效果数据库
3. **多样本测试** - 避免偶然性
4. **权衡取舍** - 准确度 vs 速度 vs 资源

### ✅ 常见错误

- ❌ 同时修改多个参数
- ❌ 只用一个音频测试
- ❌ 忽略音频质量
- ❌ 过度优化导致过拟合

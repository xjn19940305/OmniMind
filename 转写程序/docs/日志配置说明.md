# 日志配置说明

## 概述

程序已经配置了完整的日志系统，所有的输出信息都会记录到日志文件中。

## 日志文件位置

所有日志文件保存在 `logs/` 目录下：

- `funasr_YYYYMMDD.log` - 所有级别的日志（INFO, WARNING, ERROR）
- `funasr_error_YYYYMMDD.log` - 仅错误日志（ERROR）

其中 `YYYYMMDD` 是日期，例如 `funasr_20251107.log`

## 日志级别

系统使用以下日志级别：

- `INFO` - 一般信息，如服务启动、模型加载、请求处理等
- `WARNING` - 警告信息，如临时文件删除失败等
- `ERROR` - 错误信息，如模型加载失败、识别失败等
- `DEBUG` - 调试信息（默认不记录）

## 日志格式

```
2025-11-07 16:30:45 | INFO     | funasr.api | 正在启动语音识别服务...
```

格式说明：
- 时间戳：精确到秒
- 日志级别：INFO/WARNING/ERROR
- 模块名称：funasr.api（API服务）或 funasr.asr（识别引擎）
- 日志内容：具体的日志信息

## 日志轮转

- 单个日志文件最大 50MB
- 保留最近 10 个备份文件
- 自动按日期分文件
- 文件编码：UTF-8

## 查看日志

### 实时查看最新日志

```bash
# Linux/macOS
tail -f logs/funasr_$(date +%Y%m%d).log

# 查看最后 100 行
tail -n 100 logs/funasr_$(date +%Y%m%d).log
```

### 查看错误日志

```bash
# Linux/macOS
tail -f logs/funasr_error_$(date +%Y%m%d).log

# 或者从主日志文件中筛选错误
grep "ERROR" logs/funasr_*.log
```

### Windows PowerShell

```powershell
# 查看最新日志
Get-Content logs\funasr_$(Get-Date -Format "yyyyMMdd").log -Tail 50 -Wait

# 查看错误日志
Select-String "ERROR" logs\funasr_*.log
```

## 日志配置

日志配置在 `logger_config.py` 中，可以调整：

```python
# 修改日志级别
logger = setup_logger(log_level=logging.DEBUG)  # 启用调试日志

# 修改日志目录
logger = setup_logger(log_dir="custom_logs")

# 修改文件大小限制
maxBytes=100*1024*1024  # 改为100MB
```

## 生产环境建议

1. **日志清理**：定期清理旧日志文件

```bash
# 删除7天前的日志
find logs/ -name "*.log*" -mtime +7 -delete
```

2. **日志监控**：使用日志分析工具监控错误日志
   - ELK Stack (Elasticsearch + Logstash + Kibana)
   - Grafana Loki
   - 简单的脚本监控

3. **磁盘空间**：确保有足够的磁盘空间存储日志
   - 建议预留至少 10GB 空间用于日志

4. **日志级别**：生产环境建议使用 INFO 级别
   - 调试时可临时改为 DEBUG
   - 不要在生产环境长期使用 DEBUG 级别

## systemd 服务日志

如果使用 systemd 服务方式运行，还可以通过 journalctl 查看系统日志：

```bash
# 查看服务日志
sudo journalctl -u funasr -f

# 查看最近的错误
sudo journalctl -u funasr -p err

# 查看今天的日志
sudo journalctl -u funasr --since today
```

## 常见问题

### Q: 日志文件占用空间太大怎么办？

A: 可以：
1. 减小单个文件大小限制（修改 `maxBytes`）
2. 减少备份文件数量（修改 `backupCount`）
3. 定期清理旧日志
4. 使用日志压缩工具

### Q: 如何临时启用更详细的日志？

A: 修改 `logger_config.py` 中的日志级别为 DEBUG：

```python
logger = setup_logger(log_level=logging.DEBUG)
```

然后重启服务。

### Q: 日志中有乱码怎么办？

A: 确保：
1. 日志文件编码为 UTF-8（已配置）
2. 查看日志的工具支持 UTF-8 编码
3. 终端/编辑器设置为 UTF-8

### Q: 如何将日志发送到远程服务器？

A: 可以添加 HTTP Handler 或使用 syslog：

```python
# 在 logger_config.py 中添加
from logging.handlers import SysLogHandler

syslog_handler = SysLogHandler(address=('remote_host', 514))
logger.addHandler(syslog_handler)
```

## 性能影响

日志记录对性能的影响：

- **INFO 级别**：几乎无影响（< 1%）
- **DEBUG 级别**：轻微影响（< 5%）
- **文件 I/O**：异步写入，影响很小

建议在高并发环境下：
- 使用 INFO 或 WARNING 级别
- 考虑使用日志缓冲
- 将日志写入快速的磁盘（SSD）
